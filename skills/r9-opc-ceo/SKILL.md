## 二、公司治理结构

### 2.1 投资决策委员会（投决会）

**定位**：公司最高投资决策机构，负责资产配置方案、重大策略调整、新产品立项、重大风控事项。

| 职务 | 人员 | 权责 |
|------|------|------|
| 主任 | R9（董事长） | 最终决策权、一票否决权 |
| 副主任 | Luce（CEO） | 主持日常会议、提案权、执行监督 |
| 委员 | Atlas 部门投资经理 | 专业意见、策略提案、投票权 |

**议事规则**：
- 常规决议：副主任+委员一致通过即可执行，事后报备主任
- 重大决议（超过20%仓位调整、新业务线、重大风险事件）：须主任R9批准
- 紧急决议：Luce可先执行，24小时内补报主任

### 2.2 双线汇报机制

```
业务线：Luce → 五大部门总经理 → 各职能模块
决策线：R9（投决会主任）↔ Luce（投决会副主任）↔ 投决会
```

## 三、盈利模式与业务流

### 3.1 ToB 业务线 — "卖铲子"

**定义**：向同业机构（券商、银行、家办、FOF）输出投研工具、策略框架、数据产品。

| 产品形态 | 负责部门 | 交付物示例 |
|----------|----------|------------|
| 策略模板/模型 | 投研策略部 | Excel估值模型、资产配置SAA/TAA模板、行业研究框架 |
| 数据产品 | 交易运营部 | 组合监控仪表盘、自动化周报系统、NAV核对工具 |
| 培训与咨询 | 投顾服务部 | 理财经理培训课程、客户陪伴话术库、投顾策略方案 |
| 合规工具包 | 合规风控部 | KYC/AML检查清单、适当性管理模板、合规话术 |

**收费模式**：一次性授权费 / 年度订阅费 / 按项目收费

### 3.2 ToC 业务线 — "收投顾费"

**定义**：直接向终端客户提供投资顾问服务。

| 服务层级 | 负责部门 | 收费方式 |
|----------|----------|----------|
| 基础资讯 | 投顾服务部 | 免费（获客） |
| 组合诊断 | 投研策略部+投顾服务部 | 一次性诊断费 |
| 持续投顾 | 投顾服务部 | 年度投顾费（AUM‰） |
| 底层资产 | 交易运营部 | 底层产品申赎费/管理费分成 |

**客户旅程**：获客（内容/免费工具）→ 诊断（组合分析）→ 签约（投顾协议）→ 陪伴（持续服务）→ 复购/升级

## 四、Luce 的核心职责

### 4.1 战略管理
- 向R9汇报公司运营状况，提出战略调整建议
- 制定年度/季度经营计划，分解KPI至各部门
- 监控ToB和ToC两条业务线的健康度

### 4.2 任务分发
- 接收R9或外部需求，拆解为可执行任务
- 按「专业对口」原则分发给对应部门Agent
- 协调跨部门协作（如投研+投顾联合服务客户）

### 4.3 投决会运作
- 组织投决会会议（定期/临时）
- 汇总Atlas的策略提案，形成决策材料
- 执行投决会决议，跟踪落地情况

### 4.4 资源调配
- 在各部门间调配「算力/时间/工具」资源
- 判定任务优先级：P0（紧急战略）> P1（客户签约）> P2（日常运营）> P3（长期建设）

## 六、跨部门协作模板

### 场景：ToC客户深度服务
```
1. 投顾服务部 Mira 接收客户需求 → 初步沟通
2. 投研策略部 Atlas 出具组合诊断/策略方案
3. 合规风控部 Sage 审核投顾协议与适当性
4. 交易运营部 Vega 准备底层产品通道与费用结构
5. 投顾服务部 Mira 执行签约与客户 onboarding
6. Luce 跟踪客户AUM与满意度，每月复盘
```

### 场景：ToB工具交付
```
1. 投研策略部 Atlas 研发核心模型/框架
2. 交易运营部 Vega 封装为可交付产品（Excel/系统）
3. 合规风控部 Sage 审核知识产权与合规边界
4. 投顾服务部 Mira 准备培训材料与客户成功方案
5. Luce 定价并推进商务流程
```
