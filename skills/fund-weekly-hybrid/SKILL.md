# 产品分析+周度回顾融合报告

本 skill 用于生成单只基金的**融合版报告**——将基金经理横纵分析法深度评价与持营基金周度回顾合二为一。

报告以周度回顾为**主体骨架**（产品概况、全球市场速览、本周表现归因、后市展望），在"基金经理与产品档案"章节融入横纵分析法的**深度内容**（纵向职业叙事、横向竞品对比、专项选股验证、风格本质提炼）。

## 触发条件

用户以下述方式请求时触发：
- "融合版" / "产品分析+周度回顾融合"
- "横纵分析+周度回顾" / "深度周度报告"
- "给我一份既分析产品又回顾本周的"
- "基金经理深度评价+周度回顾"
- "融合版 HTML 报告" / "深度分析网页报告"
- 用户明确提到要"融合"两种分析方式

**注意**：当用户仅说"周度回顾""本周陪伴""基金周报"时，应触发 `fund-weekly-review`（纯周度回顾）。只有用户明确表达"融合""横纵""深度周度""产品分析+周度"等意图时，才触发本 skill。

## 输入变量

执行前必须明确以下变量：

```
基金代码 = 用户提供的6位基金代码（如 007119）
基金名称 = 用户提供的基金全称或简称
基金经理姓名 = 当前管理该基金的核心基金经理
报告周期 = 上周起始日期至结束日期（如 2026-06-30 至 2026-07-03）
评价基准日 = 默认当前日期
```

## 产品类型识别

生成报告前，先判断产品属于**主动权益型**还是**量化型**，后续章节根据类型切换分析框架：

| 判断维度 | 主动权益型 | 量化型 |
|---|---|---|
| 基金经理背景 | 传统研究/投资背景，擅长个股研究、产业判断 | 数学/统计/金融工程背景，擅长模型开发、因子挖掘 |
| 产品名称特征 | 无"量化""指数增强""多因子"等字样 | 含"量化""指数增强""多因子""AI策略"等字样 |
| 持仓特征 | 前十大重仓占比较高（通常>40%） | 前十大重仓占比极低（通常<10%），持仓极度分散 |
| 策略特征 | 自上而下/自下而上选股，行业配置+个股精选 | 多因子模型驱动，统计套利，量化择时/选股 |
| 换手特征 | 中低换手率（通常<200%） | 中高换手率（通常200%-500%+） |
| 业绩归因 | 个股贡献、行业配置、集中度效应 | 因子暴露、模型有效性、跟踪误差、风格漂移 |

**判定规则**：若产品名称含"量化""指数增强""多因子""AI策略"等关键词，或基金经理有数学/统计/金融工程博士背景，或前十大重仓合计<15%，则判定为**量化型**。其余判定为**主动权益型**。

## 报告结构（12个模块）

**标题格式规范**：每个模块标题使用中文数字前缀（一、二、三...），不使用emoji图标。如：一、产品概况；二、核心持仓上周表现；三、产品表现。子模块用阿拉伯数字+小数点，如：3.1 成立至今净值走势、3.2 年度业绩表、3.3 历史回撤修复能力。

> **模块编号与PDF固定模板对齐**：PDF生成器 `fund_weekly_pdf.py` 中模块顺序已硬编码，数据注入时需确保字段名称与以下模块对应。融合版与纯周度回顾共用同一模板，仅通过数据字段区分。

格式：
```
# 基金名称 · 产品分析+周度回顾

报告周期（上周起始日期 至 结束日期，如 2026-06-30 至 2026-07-03）
```

**封面指标表**（4指标）

| 最新净值 | 近一周 | 今年以来 | 成立以来 |
|---|---|---|---|
| X.XXXX | +X.XX% | +X.XX% | +X.XX% |

要求：
- 标题下方放日期范围
- 红色主调（PDF 渲染时标题和分隔线使用红色）
- 封面指标表展示最新净值、近一周收益、今年以来收益、成立以来收益

#### 3.2 量化型：核心因子与行业配置上周表现

**因子暴露表**

| 因子维度 | 暴露方向 | 上周表现 | 对净值影响 | 说明 |
|---|---|---|---|---|
| 价值因子 | 正向/负向/中性 | 因子收益 | 正向/负向 | 低估值/高估值风格上周表现 |
| 成长因子 | 正向/负向/中性 | 因子收益 | 正向/负向 | 高成长/低成长风格上周表现 |
| 动量因子 | 正向/负向/中性 | 因子收益 | 正向/负向 | 强势股/反转股上周表现 |
| 质量因子 | 正向/负向/中性 | 因子收益 | 正向/负向 | 高盈利/低盈利风格上周表现 |
| 市值因子 | 大盘/小盘/中性 | 因子收益 | 正向/负向 | 大小盘风格上周表现 |

要求：
- 不列出个股（量化产品持仓极度分散，个股意义不大），改为分析**因子暴露**和**行业配置偏离**

### 5. 四、基金经理观点

#### 5.1 主动权益型基金经理观点

**内容要求**：
- 至少4段话，每段150字以上
- 引用最新季报/年报中基金经理的"投资策略和运作分析"原文（直接引用，用引号标注）
- 如有公开访谈/路演，引用核心判断
- 阐述基金经理当前的投资策略方向（增配什么、减持什么、关注什么）
- 体现基金经理的底层投资框架和近期调仓逻辑
- **投资理念补充**：阐述基金经理的底层投资哲学（如"实业思维+精选个股+长期持有+高仓位运作"），以及当前投资理念与市场环境的契合度

#### 5.2 量化型基金经理观点

- 体现量化投资的底层逻辑（如统计套利、多因子模型、机器学习等）
- 若基金经理有公开访谈，引用其对量化策略迭代的观点
- **投资理念补充**：阐述基金经理的量化投资哲学（如"多因子模型+统计套利+严格风控"），以及当前策略与市场环境的契合度

### 10. 九、为什么上周会有波动？（归因分析）

**内容要求**：
- 4条归因分析，每条至少150字
- 正向因素编号用红色；负向因素编号用绿色
- 分维度分析上周波动来源

**主动权益型归因维度**：

1. **核心持仓端**：本基金重仓板块的上周表现（如AI算力、新能源、消费等），具体个股涨跌分析
2. **港股/海外端**：港股配置的表现，对净值的影响
3. **稳定器/拖累端**：组合中表现平稳或拖累的板块
4. **结构放大端**：高集中度/高仓位结构对波动的放大效应

每部分结尾用1-2句话小结核心原因。

---

#### 10.2 量化型归因分析

分维度分析上周波动来源，通常包括：

1. **因子暴露端**：本基金暴露的因子（价值/成长/动量/质量/市值等）上周哪些有效、哪些失效，对净值的影响
2. **行业偏离端**：模型超配/低配的行业上周表现，对净值的影响
3. **模型适配端**：当前市场环境下，量化模型的适应性如何
4. **结构放大端**：高仓位/高换手率结构对波动的放大或平滑效应

每部分结尾用1-2句话小结核心原因。

### 11. 十、后市怎么看？（后市展望）

**内容要求**：
- 4条展望，每条至少150字
- 正向因素编号用红色；负向因素编号用绿色
- 分维度展望后市

#### 11.1 主动权益型展望

分维度展望后市，通常包括：

1. **核心持仓产业链**：长期景气度判断+短期风险点
2. **港股市场**：估值空间+流动性预期
3. **其他重要板块**：如新能源、消费、医药等
4. **宏观环境**：利率、通胀、政策等

---

#### 11.2 量化型展望

分维度展望后市，通常包括：

1. **因子有效性展望**：当前市场环境下哪些因子可能持续有效
2. **行业配置展望**：模型可能超配/低配的方向
3. **模型适配性**：后续市场环境对量化策略的友好度
4. **宏观环境**：利率、流动性、政策对量化策略的影响

## 写作风格要求

1. **简洁有力**：每段不超过200字，避免冗长论述
2. **数据驱动**：每个判断都有具体数据支撑（净值、涨跌幅、占比、PE、同类排名等）
3. **专业但不晦涩**：用投资者能理解的语言解释专业判断（量化术语需适当解释）
4. **红色主调 + 红涨绿跌**：
   - PDF 渲染时标题、分隔线、表格头部使用红色系（#8b0a1a / #c41e3a）

6. **标题格式**：PDF和HTML中每个模块标题用中文数字一、二、三...前缀，不使用emoji图标。子模块用阿拉伯数字+小数点（如：3.1、3.2、3.3）
7. **类型适配**：主动权益侧重量化个股分析，量化侧重量化因子分析，不混淆
8. **融合自然**：横纵分析内容嵌入周度回顾框架，不突兀、不割裂，像"本周陪伴+深度档案"的自然组合
9. **HTML兼容**：HTML报告使用红色主调（#8b0a1a / #c41e3a），与PDF视觉统一。卡片式布局、sticky导航栏、**每5分钟自动刷新实时数据**、一键复制功能。基金经理深度档案章节（横纵分析）在网页中以折叠面板或锚点标签形式展示，保持与周度回顾模块的视觉统一

> - 两者内容同源，但 HTML 增加实时数据刷新（每5分钟）和交互功能
> - 最佳实践：邮件发送 PDF，正文附带 HTML 链接；客户可点击链接查看实时更新的网页版
> - 客户需要实时数据、手机浏览、一键复制话术、深度档案可折叠 -> 选 HTML
> - 理财经理需要打印存档、正式分发、紧凑排版 -> 选 PDF
> - 最佳实践：同时生成两种格式，PDF 用于存档，HTML 用于线上分发

```javascript
// 天天基金实时净值 JSONP（无跨域限制，最可靠）
function fetchFundNav(fundCode) {
  return new Promise((resolve) => {
    const script = document.createElement('script');
    const callbackName = '_fundgz_' + Date.now();
    window[callbackName] = (data) => {
      resolve({
        fundCode: data.fundcode,
        name: data.name,
        nav: data.dwjz,           // 单位净值（最新公布）
        navDate: data.jzrq,       // 净值日期
        estimateNav: data.gsz,    // 估算净值（实时）
        estimateChange: data.gszzl, // 估算涨跌幅%
        estimateTime: data.gztime // 估算时间
      });
      delete window[callbackName];
      document.head.removeChild(script);
    };
    script.src = `https://fundgz.1234567.com.cn/js/${fundCode}.js?rt=${Date.now()}`;
    document.head.appendChild(script);
    setTimeout(() => { resolve(null); delete window[callbackName]; }, 5000);
  });
}
```

#### 指数行情（东方财富 API — P4/P7）

```javascript
// 东方财富开放 API（支持 CORS，可直接 fetch）
async function fetchIndexQuote(secid) {
  // secid 格式：1.000300(沪深300), 0.399006(创业板), 116.HSI(恒生)等
  const url = `https://push2.eastmoney.com/api/qt/stock/get?secid=${secid}&fields=f43,f44,f45,f46,f57,f58,f60,f170,f171`;
  const res = await fetch(url);
  const json = await res.json();
  const d = json.data;
  return {
    code: d.f57, name: d.f58,
    price: (d.f43 / 100).toFixed(2),      // 当前价（分->元）
    open: (d.f46 / 100).toFixed(2),
    high: (d.f44 / 100).toFixed(2),
    low: (d.f45 / 100).toFixed(2),
    prevClose: (d.f60 / 100).toFixed(2),
    changePct: (d.f171 / 100).toFixed(2) + '%'  // 涨跌%
  };
}
```

### 5.4 异常降级策略

```javascript
// 封装带降级的数据获取函数
async function fetchWithFallback(fetchers) {
  for (const fetcher of fetchers) {
    try {
      const data = await fetcher();
      if (data && data.price !== undefined) return data;
    } catch (e) { console.warn('Source failed:', e.message); }
  }
  return null; // 全部降级失败
}

// 使用示例：获取恒生指数
const hsi = await fetchWithFallback([
  () => fetchIndexQuote('116.HSI'),      // P4 东财
  () => fetchYahooQuote('^HSI'),         // P4 Yahoo（需代理）
  () => fetchFromCache('hsi')            // P9 本地缓存兜底
]);
```

## 数据收集建议

执行研究时，优先收集以下信息：

### 主动权益型额外需要：
- 基金最新净值和上周每日净值
- 最新季报披露的前十大重仓股及行业配置
- 同花顺/东方财富的核心持仓个股上周涨跌幅
- 基金经理最新季报/年报中的"投资策略和运作分析"原文
- 基金经理近期公开访谈、路演核心观点
- 基金历史最大回撤及修复时间数据

### 量化型额外需要：
- 基金最新净值和上周每日净值
- 最新季报披露的行业配置（前5大行业即可，因持仓极度分散）
- 基金相对基准的跟踪误差、信息比率
- 近期市场风格数据（大小盘、价值成长、动量反转等因子表现）
- 基金经理关于量化策略、模型迭代的公开阐述
- 基金历史超额收益归因（如有）
- 基金历史最大回撤及修复时间数据

### 通用数据（两种类型均需）：
- 沪深300、创业板指、科创50、恒生指数、标普500上周收盘和涨跌
- 沪深300 PE、创业板指 PE、10Y国债收益率等估值指标
- 市场热门主题ETF上周表现

### 融合版额外需要：
- 基金经理的完整履历（毕业院校、入行经历、历任岗位、管理产品历史）
- 3-5只同类竞品基金的基本信息和业绩数据
- 基金经理的2-3个典型重仓股案例（买入逻辑、持有过程、验证结果）
- 同类产品的排名数据和风格标签

### 新增数据字段（写入 funds_data.json）：
- `positioning`: 产品定位说明（不少于80字）
- `holding_source`: 持仓数据来源（如"2026年一季报"）
- `holding_note`: 持仓观察（不少于100字）
- `nav_summary`: 净值走势简述（不少于150字）
- `drawdown_summary`: 回撤修复详细说明（不少于150字）
- `drawdown_repair_summary`: 回撤修复数据模块内容
- `weekly_comment`: 表现点评（不少于80字）
- `market_comment`: 市场点评（不少于50字）
- `theme_comment`: 热门主题描述（不少于100字）
- `annual_returns`: 每年为 `{"fund_return": X, "benchmark_return": X, "excess_return": X}` 对象
- `weekly_performance.periods`: 每期为 `{"period": "上周", "fund_return": X, "benchmark_return": X, "excess_return": X}`

如果数据缺失导致某些分析无法进行，应在报告中诚实说明，不虚构内容。

## 注意事项

- 不要机械复制模板中的示例文字，每个判断必须基于实际数据
- 删除不适配的模板痕迹（如"红色=上涨，绿色=下跌"等交互占位符）
- 标题下方只放日期，不要重复产品名称
- 避免每个新闻板块都用固定四段式（影响/历史/客群/话术），改为直接叙述
- 基金经理观点必须引用实际季报/访谈原文，不编造
- **主动权益与量化分析框架不混淆**：主动权益侧重量化个股，量化侧重量化因子；不强行用个股分析框架套量化产品，也不强行用因子分析框架套主动权益产品
- **横纵分析内容必须与基金经理实际经历一致**：纵向叙事的时间线、横向对比的竞品数据、选股案例的验证结果，都必须基于真实数据，不编造基金经理履历或虚构重仓股案例
- 融合版的"基金经理档案"章节控制在1.5-2页，不展开为2万字独立报告，保持与周度回顾的整体协调性
- 封面日期用日期范围（上周起始 至 结束），不显示单个日期
