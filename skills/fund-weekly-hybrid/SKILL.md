# 产品分析+周度回顾融合报告

本 skill 用于生成单只基金的**融合版报告**——将基金经理横纵分析法深度评价与持营基金周度回顾合二为一。

报告以周度回顾为**主体骨架**（产品概况、全球市场速览、本周表现归因、后市展望），在"基金经理与产品档案"章节融入横纵分析法的**深度内容**（纵向职业叙事、横向竞品对比、专项选股验证、风格本质提炼）。

## 触发条件

用户以下述方式请求时触发：
- "融合版" / "产品分析+周度回顾融合"
- "横纵分析+周度回顾" / "深度周度报告"
- "给我一份既分析产品又回顾本周的"
- "基金经理深度评价+周度回顾"
- "融合版 HTML 报告" / "深度分析网页报告"
- 用户明确提到要"融合"两种分析方式

**注意**：当用户仅说"周度回顾""本周陪伴""基金周报"时，应触发 `fund-weekly-review`（纯周度回顾）。只有用户明确表达"融合""横纵""深度周度""产品分析+周度"等意图时，才触发本 skill。

## 输入变量

执行前必须明确以下变量：

```
基金代码 = 用户提供的6位基金代码（如 007119）
基金名称 = 用户提供的基金全称或简称
基金经理姓名 = 当前管理该基金的核心基金经理
报告周期 = 本周起始日期至结束日期（如 2026-06-30 至 2026-07-03）
评价基准日 = 默认当前日期
```

## 报告结构（9个章节）

**标题格式规范**：每个章节标题使用中文数字前缀（一、二、三...），不使用emoji图标。如：一、封面标题；二、产品概况；三、核心持仓上周表现。子模块用阿拉伯数字+小数点，如：3.1 年度业绩表、3.2 历史回撤修复能力。

### 8. 为什么上周会有波动？（归因分析）

分维度分析上周波动来源，通常包括：

1. **核心持仓端**：本基金重仓板块的上周表现（如AI算力、新能源、消费等），具体个股涨跌分析
2. **港股/海外端**：港股配置的表现，对净值的影响
3. **稳定器/拖累端**：组合中表现平稳或拖累的板块
4. **结构放大端**：高集中度/高仓位结构对波动的放大效应

每部分结尾用1-2句话小结核心原因。

---

### 9. 后市怎么看？（展望）

分维度展望后市，通常包括：

1. **核心持仓产业链**：长期景气度判断+短期风险点

3. **其他重要板块**：如新能源、消费、医药等

**一、纵向职业叙事（三阶段分析）**

将基金经理的职业生涯划分为三个阶段，每个阶段提炼核心特征：

1. **第一阶段：学术奠基与早期经历**
   - 教育背景（毕业院校、专业）
   - 入行契机（为何进入金融/投资行业）
   - 早期经历（卖方/买方/研究/交易员等）
   - 核心提炼：一句话概括此阶段对投资风格的奠基作用

2. **第二阶段：积累经验与风格形成**
   - 关键岗位变动（如从研究员到基金经理助理）
   - 管理的首只产品及其特征
   - 早期投资风格（如成长/价值/均衡）
   - 核心提炼：风格在此阶段如何初步成型

3. **第三阶段：成熟管理与风格定型**
   - 当前管理产品的规模与业绩
   - 风格的成熟特征（如集中持股、高弹性、长期持有等）
   - 近期公开言论中的核心投资逻辑
   - 核心提炼：当前风格的一句话定位

**三阶段总结**：用1-2段话串联三个阶段的演化逻辑，说明基金经理是如何从早期经历走到当前管理风格的。

## 写作风格要求

1. **简洁有力**：每段不超过200字，避免冗长论述
2. **数据驱动**：每个判断都有具体数据支撑（净值、涨跌幅、占比、PE、同类排名等）
3. **专业但不晦涩**：用投资者能理解的语言解释专业判断
4. **红色主调 + 红涨绿跌**：
   - PDF 渲染时标题、分隔线、表格头部使用红色系（#8b0a1a / #c41e3a）
   - **数据颜色规则**：涨跌幅/收益率等百分比数据，正数用红色（#e74c3c / #dc2626），负数用绿色（#27ae60 / #16a34a）；净值变化同理，大于0红色，小于0绿色
   - **归因分析编号颜色**：正向因素（如核心持仓强势、稳定器贡献）编号用红色；负向因素（如拖累端、回调板块）编号用绿色
5. **紧凑排版**：页边距1.5cm，正文字号9pt，表格字号9pt，行距紧凑，目标6-8页以内
6. **标题格式**：PDF和HTML中每个章节标题用中文数字一、二、三...前缀，不使用emoji图标。子模块用阿拉伯数字+小数点（如：3.1、3.2）
7. **融合自然**：横纵分析内容嵌入周度回顾框架，不突兀、不割裂，像"本周陪伴+深度档案"的自然组合
7. **HTML兼容**：HTML报告使用红色主调（#8b0a1a / #c41e3a），与PDF视觉统一。卡片式布局、sticky导航栏、**每5分钟自动刷新实时数据**、一键复制功能。基金经理深度档案章节（横纵分析）在网页中以折叠面板或锚点标签形式展示，保持与周度回顾模块的视觉统一

### 2. PDF（红色主调，正式存档，6-8页）

### 3. HTML（红色主调，在线访问）
同时生成单页 HTML 文件，部署至 Cloudflare Pages：
- **配色**：红色主调（标题#8b0a1a、二级标题#c41e3a、卡片边框#c41e3a），**红涨绿跌**，与 PDF 视觉统一
- **布局**：sticky顶部导航标签（点击跳转模块）、卡片式模块、右上角工具栏（PDF下载/图片下载/刷新按钮）
- **交互**：一键复制、**每5分钟自动刷新实时数据**（天天基金JSONP + 东方财富API）、热门主题tab切换、横纵分析折叠面板
- **模块**：聚焦「上周」分析的10个标准模块 + 第11模块横纵分析深度档案（产品概况、核心持仓、产品表现、全球市场速览、热门主题、基金经理观点、上周产品表现、归因、后市、档案、横纵分析）
- **横纵分析展示**：在HTML中以折叠面板（details/summary）或独立标签页形式展示，点击展开详细内容，保持页面紧凑
- **文件命名**：`基金简称_融合报告.html`
- **部署路径**：`skills/portfolio-week-companion/site/reports/基金简称_融合报告.html`
- **访问地址**：`https://fundadvisor.pages.dev/reports/基金简称_融合报告.html`
- **部署方式**：Git push 到 GitHub 后，Cloudflare Pages 自动部署

> **PDF 与 HTML 的关系**：
> - PDF 是「正式文档」——用于打印、存档、邮件附件、合规留痕
> - HTML 是「在线附录」——附在 PDF 末尾，扫码或点击即可访问，用于实时数据查看、手机浏览、一键复制话术、横纵分析可折叠查看
> - 两者内容同源，但 HTML 增加实时数据刷新（每5分钟）和交互功能
> - 最佳实践：邮件发送 PDF，正文附带 HTML 链接；客户可点击链接查看实时更新的网页版
> - 客户需要实时数据、手机浏览、一键复制话术、深度档案可折叠 → 选 HTML
> - 理财经理需要打印存档、正式分发、紧凑排版 → 选 PDF
> - 最佳实践：同时生成两种格式，PDF 用于存档，HTML 用于线上分发

### 5.2 浏览器端实时数据获取（前端 JS）

对于部署到互联网（pages.dev）的静态 HTML，**浏览器端**通过以下方式获取实时数据：

#### 基金净值（天天基金 JSONP — P7）

```javascript
// 天天基金实时净值 JSONP（无跨域限制，最可靠）
function fetchFundNav(fundCode) {
  return new Promise((resolve) => {
    const script = document.createElement('script');
    const callbackName = '_fundgz_' + Date.now();
    window[callbackName] = (data) => {
      resolve({
        fundCode: data.fundcode,
        name: data.name,
        nav: data.dwjz,           // 单位净值（最新公布）
        navDate: data.jzrq,       // 净值日期
        estimateNav: data.gsz,    // 估算净值（实时）
        estimateChange: data.gszzl, // 估算涨跌幅%
        estimateTime: data.gztime // 估算时间
      });
      delete window[callbackName];
      document.head.removeChild(script);
    };
    script.src = `https://fundgz.1234567.com.cn/js/${fundCode}.js?rt=${Date.now()}`;
    document.head.appendChild(script);
    setTimeout(() => { resolve(null); delete window[callbackName]; }, 5000);
  });
}
```

#### 指数行情（东方财富 API — P4/P7）

```javascript
// 东方财富开放 API（支持 CORS，可直接 fetch）
async function fetchIndexQuote(secid) {
  // secid 格式：1.000300(沪深300), 0.399006(创业板), 116.HSI(恒生)等
  const url = `https://push2.eastmoney.com/api/qt/stock/get?secid=${secid}&fields=f43,f44,f45,f46,f57,f58,f60,f170,f171`;
  const res = await fetch(url);
  const json = await res.json();
  const d = json.data;
  return {
    code: d.f57, name: d.f58,
    price: (d.f43 / 100).toFixed(2),      // 当前价（分→元）
    open: (d.f46 / 100).toFixed(2),
    high: (d.f44 / 100).toFixed(2),
    low: (d.f45 / 100).toFixed(2),
    prevClose: (d.f60 / 100).toFixed(2),
    changePct: (d.f171 / 100).toFixed(2) + '%'  // 涨跌%
  };
}
```

| 指数 | secid |

| 恒生指数 | `116.HSI`（可能不可用）→ 降级至 `hkHSI`（新浪）或手动输入 |
| 标普500 | 需通过 Yahoo Finance 或后端代理获取 |

```javascript
// 页面加载后启动定时刷新
const REFRESH_INTERVAL = 5 * 60 * 1000; // 5分钟

async function refreshData() {
  const fundData = await fetchFundNav('025313');

setInterval(refreshData, REFRESH_INTERVAL);
```

### 5.4 异常降级策略

```javascript
// 封装带降级的数据获取函数
async function fetchWithFallback(fetchers) {
  for (const fetcher of fetchers) {
    try {
      const data = await fetcher();
      if (data && data.price !== undefined) return data;
    } catch (e) { console.warn('Source failed:', e.message); }
  }
  return null; // 全部降级失败
}

// 使用示例：获取恒生指数
const hsi = await fetchWithFallback([
  () => fetchIndexQuote('116.HSI'),      // P4 东财
  () => fetchYahooQuote('^HSI'),         // P4 Yahoo（需代理）
  () => fetchFromCache('hsi')            // P9 本地缓存兜底
]);
```

## 数据收集建议

执行研究时，优先收集以下信息：
- 基金最新净值和本周每日净值
- 最新季报披露的前十大重仓股及行业配置
- 同花顺/东方财富的核心持仓个股本周涨跌幅
- 沪深300、创业板指、科创50、恒生指数、标普500本周收盘和涨跌
- 沪深300 PE、创业板指 PE、10Y国债收益率等估值指标
- 基金经理最新季报/年报中的"投资策略和运作分析"原文
- 基金经理近期公开访谈、路演核心观点
- 市场热门主题ETF本周表现
- 基金历史最大回撤及修复时间数据
- **融合版额外需要**：
  - 基金经理的完整履历（毕业院校、入行经历、历任岗位、管理产品历史）
  - 3-5只同类竞品基金的基本信息和业绩数据
  - 基金经理的2-3个典型重仓股案例（买入逻辑、持有过程、验证结果）
  - 同类产品的排名数据和风格标签

如果数据缺失导致某些分析无法进行，应在报告中诚实说明，不虚构内容。

## 注意事项

- 不要机械复制模板中的示例文字，每个判断必须基于实际数据
- 删除不适配的模板痕迹（如"红色=上涨，绿色=下跌"等交互占位符）
- 标题下方只放日期，不要重复产品名称
- 避免每个新闻板块都用固定四段式（影响/历史/客群/话术），改为直接叙述
- 基金经理观点必须引用实际季报/访谈原文，不编造
- **横纵分析内容必须与基金经理实际经历一致**：纵向叙事的时间线、横向对比的竞品数据、选股案例的验证结果，都必须基于真实数据，不编造基金经理履历或虚构重仓股案例
- 融合版的"基金经理档案"章节控制在1.5-2页，不展开为2万字独立报告，保持与周度回顾的整体协调性
