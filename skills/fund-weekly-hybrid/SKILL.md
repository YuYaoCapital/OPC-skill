# 产品分析+周度回顾融合报告

本 skill 用于生成单只基金的**融合版报告**——将基金经理横纵分析法深度评价与持营基金周度回顾合二为一。

报告以周度回顾为**主体骨架**（产品概况、全球市场速览、本周表现归因、后市展望），在"基金经理与产品档案"章节融入横纵分析法的**深度内容**（纵向职业叙事、横向竞品对比、专项选股验证、风格本质提炼）。

## 触发条件

用户以下述方式请求时触发：
- "融合版" / "产品分析+周度回顾融合"
- "横纵分析+周度回顾" / "深度周度报告"
- "给我一份既分析产品又回顾本周的"
- "基金经理深度评价+周度回顾"
- "融合版 HTML 报告" / "深度分析网页报告"
- 用户明确提到要"融合"两种分析方式

**注意**：当用户仅说"周度回顾""本周陪伴""基金周报"时，应触发 `fund-weekly-review`（纯周度回顾）。只有用户明确表达"融合""横纵""深度周度""产品分析+周度"等意图时，才触发本 skill。

## 输入变量

执行前必须明确以下变量：

```
基金代码 = 用户提供的6位基金代码（如 007119）
基金名称 = 用户提供的基金全称或简称
基金经理姓名 = 当前管理该基金的核心基金经理
报告周期 = 本周起始日期至结束日期（如 2026-06-30 至 2026-07-03）
评价基准日 = 默认当前日期
wechat_webhook = 企业微信机器人 webhook URL（可选；提供后将自动推送报告到企微群）
```

- 当前波动作为最后一行

### 7. 为什么本周会有波动？（归因分析）

分维度分析本周波动来源，通常包括：

1. **核心持仓端**：本基金重仓板块的本周表现（如AI算力、新能源、消费等），具体个股涨跌分析
2. **港股/海外端**：港股配置的表现，对净值的影响
3. **稳定器/拖累端**：组合中表现平稳或拖累的板块
4. **结构放大端**：高集中度/高仓位结构对波动的放大效应

每部分结尾用1-2句话小结核心原因。

### 8. 后市怎么看？（展望）

分维度展望后市，通常包括：

1. **核心持仓产业链**：长期景气度判断+短期风险点
2. **港股市场**：估值空间+流动性预期
3. **其他重要板块**：如新能源、消费、医药等
4. **宏观环境**：利率、通胀、政策等

**一、纵向职业叙事（三阶段分析）**

将基金经理的职业生涯划分为三个阶段，每个阶段提炼核心特征：

1. **第一阶段：学术奠基与早期经历**
   - 教育背景（毕业院校、专业）
   - 入行契机（为何进入金融/投资行业）
   - 早期经历（卖方/买方/研究/交易员等）
   - 核心提炼：一句话概括此阶段对投资风格的奠基作用

2. **第二阶段：积累经验与风格形成**
   - 关键岗位变动（如从研究员到基金经理助理）
   - 管理的首只产品及其特征
   - 早期投资风格（如成长/价值/均衡）
   - 核心提炼：风格在此阶段如何初步成型

3. **第三阶段：成熟管理与风格定型**
   - 当前管理产品的规模与业绩
   - 风格的成熟特征（如集中持股、高弹性、长期持有等）
   - 近期公开言论中的核心投资逻辑
   - 核心提炼：当前风格的一句话定位

**三阶段总结**：用1-2段话串联三个阶段的演化逻辑，说明基金经理是如何从早期经历走到当前管理风格的。

**对比分析要点**：

- 本产品的相对劣势（如规模过大导致灵活性下降、港股配置不足等）

**三、专项选股验证（2-3个案例）**

选取基金经理的2-3个典型重仓股案例，验证其投资逻辑与执行力：

**案例1：股票名称**
- 买入逻辑（为什么选这只股票？产业判断？公司质地？）
- 持有过程（何时买入、仓位变化、是否承受过波动）
- 验证结果（该股票表现如何，是否兑现了投资逻辑）

**案例2、案例3**：同上结构

**选股能力总结**：1-2句话概括基金经理的选股特征（如：擅长左侧布局成长股、偏好产业龙头、敢于高集中度持有等）

**四、横纵交汇（风格本质提炼）**

> 例："这是一位从卖方研究员转型而来的成长型基金经理，经历了A股多轮牛熊后形成了'高集中度+高弹性+长期持有'的投资风格，在同类产品中属于进攻性最强的一档，但也因此面临较大的波动风险。"

## 写作风格要求

1. **简洁有力**：每段不超过200字，避免冗长论述
2. **数据驱动**：每个判断都有具体数据支撑（净值、涨跌幅、占比、PE、同类排名等）
3. **专业但不晦涩**：用投资者能理解的语言解释专业判断
4. **红色主调**：PDF 渲染时标题、分隔线、表格头部使用红色系
5. **紧凑排版**：页边距1.5cm，正文字号9pt，表格字号9pt，行距紧凑，目标6-8页以内
6. **融合自然**：横纵分析内容嵌入周度回顾框架，不突兀、不割裂，像"本周陪伴+深度档案"的自然组合
7. **HTML兼容**：HTML报告使用与 portfolio-week-companion 相同的深蓝主调（#1e3a5f）、卡片式布局、sticky导航栏、实时数据刷新、一键复制功能。基金经理深度档案章节（横纵分析）在网页中以折叠面板或锚点标签形式展示，保持与周度回顾模块的视觉统一

## 输出格式

**标准交付：同时生成 PDF + HTML 两种格式**，PDF 作为正式存档/分发文档，HTML 作为在线可访问的网页附录。

### 1. Markdown（初稿）
先生成 Markdown 文件（.md），作为内容初稿和校对基础。文件命名：`YYYYMMDD_基金简称_融合报告.md`。

### 2. PDF（红色主调，正式存档）
使用 reportlab 生成，要求：
- 中文字体使用 SimHei（黑体）
- **红色主调**（标题#8b0a1a、二级标题#c41e3a、表格头部红色底白字）
- 表格内容居中、字体9pt、行高紧凑
- 列宽自适应（内容多的列更宽，如"说明"列；内容少的列更窄，如"涨跌"列）
- 6-8页以内（比纯周度回顾略长，但比独立深度评价大幅精简）
- **附录页**：在PDF末尾附上一页「HTML在线报告」，包含二维码和访问链接 `https://fundadvisor.pages.dev/reports/YYYYMMDD_基金简称_融合报告.html`

### 3. HTML（红色主调，在线访问）
同时生成单页 HTML 文件，部署至 Cloudflare Pages：
- **配色**：红色主调（标题#8b0a1a、二级标题#c41e3a、卡片边框#c41e3a），红涨绿跌，与 PDF 视觉统一
- **布局**：sticky顶部导航标签（点击跳转模块）、卡片式模块、右上角工具栏
- **交互**：一键复制、实时数据刷新（天天基金JSONP + 东方财富API）、热门主题tab切换、横纵分析折叠面板
- **模块**：聚焦「上周」分析的9个标准模块 + 第10模块横纵分析深度档案（纵向职业叙事、横向竞品对比、专项选股验证、风格本质提炼）
- **横纵分析展示**：在HTML中以折叠面板（details/summary）或独立标签页形式展示，点击展开详细内容
- **文件命名**：`YYYYMMDD_基金简称_融合报告.html`
- **部署路径**：`skills/portfolio-week-companion/site/reports/YYYYMMDD_基金简称_融合报告.html`
- **访问地址**：`https://fundadvisor.pages.dev/reports/YYYYMMDD_基金简称_融合报告.html`
- **部署方式**：Git push 到 GitHub 后，Cloudflare Pages 自动部署

> **PDF 与 HTML 的关系**：
> - PDF 是「正式文档」——用于打印、存档、邮件附件、合规留痕
> - HTML 是「在线附录」——附在 PDF 末尾，扫码或点击即可访问，用于实时数据查看、手机浏览、一键复制话术、横纵分析可折叠查看
> - 两者内容同源，但 HTML 增加实时数据刷新和交互功能
> - 最佳实践：邮件发送 PDF，正文附带 HTML 链接；客户可点击链接查看实时更新的网页版

### 4. 推送到企业微信群（可选）

若用户提供了 `wechat_webhook`，在 PDF 与 HTML 生成完成后，调用 `scripts/send_wechat_work.py` 将报告推送到企业微信群。

推送内容：
- **Markdown 消息**：报告标题、基金代码/名称、报告日期、HTML 在线访问链接
- **PDF 附件**：通过企业微信 `upload_media` + `file` 消息类型上传并发送
- **HTML 附件**（可选）：如需同时发送 HTML 文件，追加 `--html-as-file` 参数

调用示例：

```bash
python skills/fund-weekly-hybrid/scripts/send_wechat_work.py \
  --webhook "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY" \
  --title "007119 产品分析+周度回顾融合报告" \
  --fund-code "007119" \
  --fund-name "睿远成长价值混合A" \
  --date "2026-06-30 至 2026-07-03" \
  --pdf "reports/20260703_睿远成长价值_融合报告.pdf" \
  --html "https://fundadvisor.pages.dev/reports/20260703_睿远成长价值_融合报告.html"
```

参数说明：

| 参数 | 必填 | 说明 |
|---|---|---|
| `--webhook` | 是 | 企业微信机器人 webhook URL |
| `--title` | 是 | 报告标题，会作为 markdown 消息首行 |
| `--fund-code` | 否 | 基金代码 |
| `--fund-name` | 否 | 基金名称 |
| `--date` | 否 | 报告日期 |
| `--pdf` | 否 | PDF 报告本地文件路径 |
| `--html` | 否 | HTML 在线链接或本地文件路径 |
| `--html-as-file` | 否 | 同时将 HTML 作为文件附件上传 |
| `--extra` | 否 | 附加说明文本 |
| `--dry-run` | 否 | 仅打印待发送内容，不实际推送 |

> **注意事项**：
> - 企业微信群机器人对文件大小有限制（通常 ≤ 20MB），超大 PDF 需先压缩
> - webhook URL 中的 `key` 为群机器人唯一凭证，不要硬编码在公开仓库中
> - 若只需要发送在线 HTML 链接，不提供 `--pdf` 即可
