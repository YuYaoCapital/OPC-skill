# 持营基金周回顾

本 skill 用于生成单只基金的**纯周度陪伴/回顾报告**（不含横纵分析法深度评价）。报告融合产品概况、全球市场速览、热门主题、基金经理观点、本周业绩表现与归因、后市展望以及简要的基金经理档案，输出为红色主调的 Markdown 或 PDF 报告（5页以内）。

> 如需**融合版**（同时包含横纵分析法深度评价的基金经理档案），请使用 `fund-weekly-hybrid` skill。

## 触发条件

用户以下述方式请求时触发：
- "生成 XX 基金的周度回顾"
- "帮我写一份 XX 基金周度陪伴"
- "持营基金周报告"
- "基金周度回顾"
- "本周基金表现分析"
- "基金周报"
- "生成 HTML 周度回顾" / "网页版基金周报"
- "pages.dev 基金报告" / "在线基金周报"

**注意**：当用户明确提到"融合版""横纵分析""深度周度""产品分析+周度回顾""既分析产品又回顾本周"时，应触发 `fund-weekly-hybrid`（融合版），而非本 skill。

## 输入变量

执行前必须明确以下变量：

```
基金代码 = 用户提供的6位基金代码（如 007119）
基金名称 = 用户提供的基金全称或简称
基金经理姓名 = 当前管理该基金的核心基金经理
报告日期 = 上周最后一个交易日日期（如 2026-07-03）
产品类型 = 主动权益 / 量化（默认主动权益，如基金经理有量化背景或产品名称含"量化""指数增强""多因子"等则判定为量化）
评价基准日 = 默认当前日期
```

## 产品类型识别

生成报告前，先判断产品属于**主动权益型**还是**量化型**，后续章节根据类型切换分析框架：

| 判断维度 | 主动权益型 | 量化型 |
|---|---|---|
| 基金经理背景 | 传统研究/投资背景，擅长个股研究、产业判断 | 数学/统计/金融工程背景，擅长模型开发、因子挖掘 |
| 产品名称特征 | 无"量化""指数增强""多因子"等字样 | 含"量化""指数增强""多因子""AI策略"等字样 |
| 持仓特征 | 前十大重仓占比较高（通常>40%） | 前十大重仓占比极低（通常<10%），持仓极度分散 |
| 策略特征 | 自上而下/自下而上选股，行业配置+个股精选 | 多因子模型驱动，统计套利，量化择时/选股 |
| 换手特征 | 中低换手率（通常<200%） | 中高换手率（通常200%-500%+） |
| 业绩归因 | 个股贡献、行业配置、集中度效应 | 因子暴露、模型有效性、跟踪误差、风格漂移 |

**判定规则**：若产品名称含"量化""指数增强""多因子""AI策略"等关键词，或基金经理有数学/统计/金融工程博士背景，或前十大重仓合计<15%，则判定为**量化型**。其余判定为**主动权益型**。

## 报告结构（从上到下，10个模块）

**标题格式规范**：每个模块标题使用中文数字前缀（一、二、三...），不使用emoji图标。如：一、产品概况；二、核心持仓上周表现；三、产品表现。子模块用阿拉伯数字+小数点，如：3.1 成立至今净值走势、3.2 年度业绩、3.3 历史回撤修复能力。

### 9. 为什么上周会有波动？（归因分析）

### 8. 为什么上周会有波动？（归因分析）

**内容要求**：
- 4条归因分析，每条至少150字
- 正向因素编号用红色；负向因素编号用绿色
- 分维度分析上周波动来源

**主动权益型归因维度**：

1. **核心持仓端**：本基金重仓板块的上周表现（如AI算力、新能源、消费等），具体个股涨跌分析
2. **港股/海外端**：港股配置的表现，对净值的影响
3. **稳定器/拖累端**：组合中表现平稳或拖累的板块
4. **结构放大端**：高集中度/高仓位结构对波动的放大效应

每部分结尾用1-2句话小结核心原因。

---

#### 9.2 量化型归因分析

分维度分析上周波动来源，通常包括：

1. **因子暴露端**：本基金暴露的因子（价值/成长/动量/质量/市值等）上周哪些有效、哪些失效，对净值的影响
2. **行业偏离端**：模型超配/低配的行业上周表现，对净值的影响
3. **模型适配端**：当前市场环境下，量化模型的适应性如何
4. **结构放大端**：高仓位/高换手率结构对波动的放大或平滑效应

每部分结尾用1-2句话小结核心原因。

### 9. 后市怎么看？（后市展望）

**内容要求**：
- 4条展望，每条至少150字
- 正向因素编号用红色；负向因素编号用绿色
- 分维度展望后市

#### 10.1 主动权益型展望

分维度展望后市，通常包括：

1. **核心持仓产业链**：长期景气度判断+短期风险点
2. **港股市场**：估值空间+流动性预期
3. **其他重要板块**：如新能源、消费、医药等
4. **宏观环境**：利率、通胀、政策等

---

#### 10.2 量化型展望

分维度展望后市，通常包括：

1. **因子有效性展望**：当前市场环境下哪些因子可能持续有效
2. **行业配置展望**：模型可能超配/低配的方向
3. **模型适配性**：后续市场环境对量化策略的友好度
4. **宏观环境**：利率、流动性、政策对量化策略的影响

## 写作风格要求

1. **简洁有力**：每段不超过200字，避免冗长论述
2. **数据驱动**：每个判断都有具体数据支撑（净值、涨跌幅、占比、PE、因子收益等）
3. **专业但不晦涩**：用投资者能理解的语言解释专业判断（量化术语需适当解释）
4. **红色主调 + 红涨绿跌**：
   - PDF 渲染时标题、分隔线、表格头部使用红色系（#8b0a1a / #c41e3a）
   - **数据颜色规则**：涨跌幅/收益率等百分比数据，正数用红色（#e74c3c / #dc2626），负数用绿色（#27ae60 / #16a34a）；净值变化同理，大于0红色，小于0绿色
   - **归因分析编号颜色**：正向因素（如核心持仓强势、稳定器贡献）编号用红色；负向因素（如拖累端、回调板块）编号用绿色
5. **紧凑排版**：页边距1.5cm，正文字号9pt，表格字号9pt，行距紧凑，目标5页以内
6. **标题格式**：PDF和HTML中每个模块标题用中文数字一、二、三...前缀，不使用emoji图标。子模块用阿拉伯数字+小数点（如：3.1、3.2、3.3）
7. **类型适配**：主动权益侧重量化个股分析，量化侧重量化因子分析，不混淆
7. **HTML兼容**：HTML报告样式完全参考融合报告（排除横纵分析深度档案部分），使用红色主调（#8b0a1a / #c41e3a），与PDF视觉统一。卡片式布局、sticky导航栏、一键复制功能。确保PDF与HTML两种输出格式视觉统一

```javascript
// 天天基金实时净值 JSONP（无跨域限制，最可靠）
function fetchFundNav(fundCode) {
  return new Promise((resolve) => {
    const script = document.createElement('script');
    const callbackName = '_fundgz_' + Date.now();
    window[callbackName] = (data) => {
      resolve({
        fundCode: data.fundcode,
        name: data.name,
        nav: data.dwjz,           // 单位净值（最新公布）
        navDate: data.jzrq,       // 净值日期
        estimateNav: data.gsz,    // 估算净值（实时）
        estimateChange: data.gszzl, // 估算涨跌幅%
        estimateTime: data.gztime // 估算时间
      });
      delete window[callbackName];
      document.head.removeChild(script);
    };
    script.src = `https://fundgz.1234567.com.cn/js/${fundCode}.js?rt=${Date.now()}`;
    document.head.appendChild(script);
    setTimeout(() => { resolve(null); delete window[callbackName]; }, 5000);
  });
}
```

#### 指数行情（东方财富 API — P4/P7）

```javascript
// 东方财富开放 API（支持 CORS，可直接 fetch）
async function fetchIndexQuote(secid) {
  // secid 格式：1.000300(沪深300), 0.399006(创业板), 116.HSI(恒生)等
  const url = `https://push2.eastmoney.com/api/qt/stock/get?secid=${secid}&fields=f43,f44,f45,f46,f57,f58,f60,f170,f171`;
  const res = await fetch(url);
  const json = await res.json();
  const d = json.data;
  return {
    code: d.f57, name: d.f58,
    price: (d.f43 / 100).toFixed(2),      // 当前价（分→元）
    open: (d.f46 / 100).toFixed(2),
    high: (d.f44 / 100).toFixed(2),
    low: (d.f45 / 100).toFixed(2),
    prevClose: (d.f60 / 100).toFixed(2),
    changePct: (d.f171 / 100).toFixed(2) + '%'  // 涨跌%
  };
}
```

### 5.4 异常降级策略

```javascript
// 封装带降级的数据获取函数
async function fetchWithFallback(fetchers) {
  for (const fetcher of fetchers) {
    try {
      const data = await fetcher();
      if (data && data.price !== undefined) return data;
    } catch (e) { console.warn('Source failed:', e.message); }
  }
  return null; // 全部降级失败
}

// 使用示例：获取恒生指数
const hsi = await fetchWithFallback([
  () => fetchIndexQuote('116.HSI'),      // P4 东财
  () => fetchYahooQuote('^HSI'),         // P4 Yahoo（需代理）
  () => fetchFromCache('hsi')            // P9 本地缓存兜底
]);
```
