# 每日市场热点解读（R9风格 × R9横纵分析法）

## 用途

将当日市场新闻、行情数据、公司公告、政策变化等素材，转化为一篇风格鲜明、
可读性强、兼具网感与深度的财经解读文章。文章以网络化、口语化的外壳包裹
R9横纵分析法的内核，适合财经自媒体每日发布。

## 触发场景

- "帮我写今天的市场点评"
- "生成一篇R9风格的市场解读"
- "今日财经早评"
- "收盘点评怎么写"
- "把今天的市场新闻写成文章"
- "模仿喵点睛风格写市场分析"

## 工作流程

### 第一步：收集素材

用户提供当日素材，包括但不限于：

1. **昨日/当日行情数据**：指数涨跌幅、成交额、涨跌家数、主要板块表现
2. **核心新闻**：政策、地缘、产业动态、公司重大事件
3. **公司公告**：重要业绩、增减持、IPO进展、并购重组
4. **全球市场**：隔夜美股、欧股、A50、原油、黄金等表现
5. **可选素材**：研报观点、机构动向、数据图表等

> 用户可直接粘贴新闻，或提供链接由工具抓取。

### 第二步：确定主线与标题

**标题规则**：
- 简短有力，通常不超过10个字
- 带情绪色彩，口语化
- 可用省略号".."、问号"?"、感叹号"!"增强语气
- 例："消失的老登。"、"突发，谈崩了？！"、"踏空了怎么办"、"恒科飙了。"、"亮瞎眼了。"

**主线判断**：
- 识别当日市场最核心的1-2个矛盾或事件
- 常见框架：科技（小登）vs 传统（老登）、地缘博弈、政策变化、产业突破

### 第三步：撰写文章

按照以下固定结构输出，**必须严格遵守语言风格规范**：

#### 结构模板

```markdown
## 【标题】

### 今日聚焦

#### 市场综述
[昨日/当日市场整体表现：指数涨跌、成交额变化、涨跌家数，100-200字]

#### 核心矛盾
[当日最突出的1-2个市场矛盾或事件深度解读，300-600字]
[如涉及极端数据，在此处插入深度历史复盘]

---

#### 重点新闻

[用自然过渡词串联多条新闻，形成流畅叙述，避免编号罗列。每段聚焦一个主题，段与段之间用过渡句衔接。]

---

#### 全球市场

[隔夜美股/全球市场简要回顾，50-100字]

数据来源：choice
```

#### 语言风格规范（核心）

**1. 网络化代称（必须熟练使用）**

| 代称 | 含义 | 使用场景 |
|------|------|---------|
| 老登 | 传统行业/板块（白酒、消费、地产、电力、银行、医药等） | "老登又挨打了"、"老登回了一滴血" |
| 小登 | 科技成长方向（AI、半导体、芯片、机器人、光模块等） | "小登血脉觉醒"、"小登狂飙" |
| 登王 | 老登中的龙头板块，通常指白酒 | "登王白酒大涨3%" |
| 疯王 | 特朗普 | "疯王又甩出大棒" |
| 朗子 | 伊朗 | "朗子态度强硬" |
| 鹅厂 | 腾讯 | "鹅厂推出AI助手" |
| 迪子 | 比亚迪 | "迪子布局人形机器人" |
| 阿科 | 万科 | "阿科一度涨停" |
| 茅子 | 茅台 | "茅子卡在整数点位止跌" |

**2. 语气与句式**

- **短句为主**：每句话控制在20字以内，多使用逗号、句号分割
- **省略号常态化**：用".."".."表示停顿、无语、意味深长
- **口语化连接**："说白了"、"其实"、"不过客观来说"、"言归正传"、"只能说"
- **情绪词点缀**："蚌埠住了"、"亮瞎眼了"、"真的猛"、"挺扯的"、"够呛"、"够狠"
- **自我调侃**："嘴炮预测一波"、"看看这次蒙的准不准"
- **设问反问**："是不打算谈了？"、"真的能改变世界？"

**3. 分析逻辑（融入R9横纵分析法）**

每日市场点评不只是"今天发生了什么"，而是要回答**"为什么发生、接下来会怎样"**。借鉴R9横纵分析法的核心思想，文章要有时间纵深（纵向）和横向对比（横向），最终交汇出判断。

**纵向：时间纵深——事件是怎么走到今天的**
- **追溯因果链**：不是简单陈述"博通跌12%导致芯片股大跌"，而是追问——博通怎么一步步成为AI定制芯片霸主的？市场如何把预期一步步打满的？这次miss为什么是压垮骆驼的最后一根稻草？
- **还原决策逻辑**：资金为什么在这个时间点选择卖出？是获利盘兑现、还是趋势逆转的信号？背后的真实动机是什么？
- **用故事叙事**：把冷冰冰的行情数据还原成有前因后果的市场故事，让读者感受到"这件事不是突然发生的，而是酝酿已久"

**横向：同期对比——同类资产在横向切面上的分化**
- **跨市场对比**：A股芯片 vs 美股芯片 vs 韩国芯片，跌幅差异说明了什么？（如A股跌更多是因为前期涨更多、估值更贵）
- **跨板块对比**：小登（科技）vs 老登（传统），资金在两者之间如何博弈？有没有"燃烧老登、点亮科技"的跷跷板效应？
- **跨资产对比**：芯片股跌的时候，资金去了哪里？是去了防御性板块（煤炭、电力），还是去了债券/黄金等避险资产？
- **对比要有温度**：不是罗列涨跌幅表格，而是讲清楚每个方向"活成了什么样"——半导体为什么扛不住？煤炭为什么能撑住？资金选它的真实理由是什么？

**横纵交汇：综合判断**
- 把纵向（历史修复节奏）和横向（全球联动格局）结合起来
- 给出综合判断：这次和历史案例有什么本质不同？当前有什么变量是以前没有的？
- 例："纵向看，历史上科创50急跌后通常1-5个交易日修复；横向看，这次是全球同步下跌（美股、韩股、日股一起跌），修复节奏可能慢于5月底但快于4月底，因为..."

**基础原则（保持不变）**
- **先说现象，再给观点**：不绕弯子，直接亮明态度
- **数据+逻辑+观点**：引用数据时简洁，重点在解读
- **风险提示不缺席**：即使看多也会提醒"注意节奏和分寸，不要太上头"
- **鲜明不模棱两可**：避免"或有可能""一定程度上"等模糊表达，用"几乎明牌"
"大概率""毋庸置疑"等确定性词汇
- **叙事驱动，不是罗列驱动**：用故事的方式把市场变化串起来，有起承转合，有因果链条

**4. 新闻串联规范（重要，避免罗列）**

- **禁用纯编号罗列**：不要写"1、xxx 2、xxx 3、xxx"
- **用过渡词自然衔接**：
  - 递进："除此之外"、"更夸张的是"、"雪上加霜的是"
  - 转折："不过话说回来"、"但另一边"、"相比之下"
  - 补充："还有一个细节值得注意"、"有意思的是"、"别忘了"
  - 因果："受这事拖累"、"受此影响"、"说白了就是"
- **分段原则**：每段聚焦一个主题，段内讲清楚来龙去脉，段间用过渡句衔接
- **小标题使用**：只在"今日聚焦"和"重点新闻"这两个大模块前用`####`，新闻内部不再加小标题

> ✅ 示例：
> 
> 消息面上，全球芯片市场的情绪隔夜被博通给带崩了。博通二季度营收虽然创了纪录，但AI芯片业务的指引低于市场预期，盘后股价直接跳水，最终暴跌超12%...直接动摇了市场对AI基建景气度的信仰。
> 
> 受这事拖累，美股芯片股周四全遭抛售，费城半导体指数重挫5%，美光大跌近8%...韩国也没跑掉，综指一度跌超6%，KOSPI 200期货直接触发熔断...这种级别的波动，说明全球资金对科技股的持仓已经高度拥挤。
> 
> 除此之外，日本加息预期也在持续升温。日本央行行长植田和男近期表态偏向鹰派，若16日真的落地...不过目前市场对此已经有所预期，真落地了杀伤力可能有限。
> 
> 相比之下，国内倒是有个相对暖心的消息。房地产贷款"五档两线"管理制度已进一步放松...若地产能持续复苏，对整个股市都会有非常积极的影响。

**5. 段落风格**

- 每个段落3-5句话，不超过100字
- 段与段之间空一行
- 今日聚焦部分：先写市场整体，再写板块分化，再深入到核心事件
- 重点新闻部分：每段一个主题，过渡词衔接，读起来像讲故事而不是读公告

**6. 深度历史复盘规范**

**触发条件**（满足任一即触发）：
- 主要指数单日涨跌幅 ≥ 3%
- 板块单日涨跌幅 ≥ 5%
- 成交额创近30日新高/新低，或较前一日缩量/放量超20%
- 个股/板块估值突破近5年极值（如PE分位点>95%或<5%）
- 重大政策/地缘事件首次出现，或强度超预期

**复盘维度**（至少包含2-3项）：
- **历史上哪几次出现过？** 具体时间、背景
- **当时的前因后果？** 是什么触发的，后续如何演变
- **后续N天走势？** 第1天、第3天、第5天、第10天的涨跌幅
- **修复花了多久？** 几天止跌、几天回到前高、修复幅度多少
- **当前与历史案例的异同？** 宏观环境、估值水平、资金结构有何不同
- **结论**：历史会简单重复吗？当前有什么变量不同

**写作要求**：
- 历史数据必须准确，不确定的宁可不写具体数字，写"此前几次类似情况"
- 对比要有意义，不要硬凑
- 结尾要落到"对当下投资的启示"，不能光罗列历史

> ✅ 示例：
> 
> "科创50单日跌4%，这种级别的急杀跌在今年不是第一次。上次出现是5月29日，当天跌了5%，起因也是外围芯片股暴跌。但那次只调了1个交易日，次日便大幅反弹3.2%，3个交易日内就收复了失地。再往前看，1月27日DeepSeek冲击那次，科创50跌了3.8%，后续花了5个交易日才修复。之所以修复节奏不同，核心区别在于当时的估值位置和资金结构——5月底科技板块刚经历一波调整，获利盘不多，承接力强；而1月时处于高位，筹码松动更明显。眼下这个位置，估值比5月底高、但比1月底低，修复时间大概率介于两者之间。"

**7. 数据引用规范**

- 引用数据时简洁，不要堆砌数字
- 关键数据用口语化方式带出，如"成交额放大到了3.15万亿"、"科创50单日跌4%"
- 纯文字叙述即可，用户如需做图可基于文章自行提炼数据

**8. 常用表达库**

| 场景 | 常用表达 |
|------|---------|
| 市场分化 | "K型走势进行到底"、"一九分化"、"割裂"、"撕裂" |
| 科技上涨 | "小登狂飙"、"血脉觉醒"、"亮瞎眼了"、"涨停潮" |
| 传统下跌 | "老登挨打"、"失血"、"阴包阳"、"被虹吸" |
| 高位震荡 | "筹码松动"、"风吹草动都要重视"、"获利盘丰厚" |
| 地缘博弈 | "反复扯皮"、"各说各话"、"折腾够呛" |
| 配置建议 | "几乎明牌的简单题"、"底仓配置"、"哑铃策略" |
| 历史对比 | "历史上类似的..."、"上次出现这种情况是..."、"复盘来看..."、"花了几日修复" |
| 递进衔接 | "除此之外"、"更夸张的是"、"雪上加霜的是" |
| 转折衔接 | "不过话说回来"、"但另一边"、"相比之下" |
| 补充衔接 | "还有一个细节值得注意"、"有意思的是"、"别忘了" |
| 因果衔接 | "受这事拖累"、"受此影响"、"说白了就是" |
| 结尾 | "就这些了"、"边走边看"、"保持跟踪" |

### 第四步：数据校验（强制）

文章写完后，**必须逐条核对**文中出现的所有数据：

- [ ] **行情数据**：指数涨跌幅、点位、成交额是否与用户提供的素材一致？
- [ ] **个股数据**：涨跌幅、市值变化、PE/PB等是否有误？
- [ ] **宏观数据**：GDP、CPI、PMI、失业率等是否准确？
- [ ] **历史对比数据**：检索到的历史案例时间、幅度、后续走势是否准确？
- [ ] **换算数据**："蒸发X亿美元"、"折合人民币X万亿"等换算是否正确？
- [ ] **时间线**：事件发生的先后顺序是否搞混？（尤其注意隔夜美股 vs 当日A股）

**校验规则**：
- 若发现数据存疑，**优先向用户确认**，不要猜
- 若无法确认，**用模糊表述替代具体数字**（如"跌幅接近X%"改为"跌幅较大"）
- 绝对禁止编造具体数字

### 第五步：内容安全审查（强制，针对公众号发布）

文章定稿前，**必须逐句审查**是否存在以下敏感风险，避免发出去被删帖或限流：

**高风险（必须替换或删除）**：

| 敏感类型 | 示例 | 建议替换为 |
|---------|------|-----------|
| 政治人物负面代称 | "疯王"、"登子"、"睡王" | "美方"、"某人"、"老特"（较低风险）、直接用姓名 |
| 血腥暴力词汇 | "血洗"、"屠杀"、"杀疯了" | "重挫"、"大幅调整"、"剧烈波动"、"集体跳水" |
| 渲染战争/冲突细节 | 精确描述军事打击、导弹型号、伤亡数字 | 简化为"地缘摩擦"、"双方博弈"、"局势升级" |
| 对政策的讽刺性解读 | "挺扯的"、"真魔幻"（针对国内政策） | 保留对海外的调侃，国内政策用中性词 |
| 恐慌性金融词汇 | "崩盘"、"股灾"、"完蛋了" | "深度调整"、"剧烈震荡"、"承压明显" |
| 疫情/公共卫生关联 | 将市场下跌归因于疫情 | 避免提及 |

**中风险（建议谨慎使用）**：

| 敏感类型 | 示例 | 处理建议 |
|---------|------|---------|
| 宗教/民族相关 | "伊斯兰"、"犹太"、"阿拉伯"等具体指向 | 用国家名替代（如"伊朗"代替"伊斯兰"），避免宗教定性 |
| 核武器/拥核 | "拥核"、"核武"、"铀浓缩" | 简化为"核问题"、"相关协议" |
| 社会群体性事件 | 罢工、游行、集会 | 简化为"当地局势"或删除 |
| 金融机构具体负面 | "某某基金爆仓"、"某券商踩雷" | 如无确切信源，删除 |

**审查原则**：
- 宁可保守，不要冒险。不确定的词一律替换。
- 海外政治人物可以调侃，但**不要指名道姓+负面标签组合**（如"疯王+特朗普"同时出现）。
- 地缘新闻**点到为止**，不要展开军事细节。
- 对国内政策/监管**绝对中性**，不调侃、不讽刺。
- 遇到重大敏感事件（如突发地缘冲突升级），**主动降低情绪浓度**，少用电击性词汇。

### 第六步：检查与交付

- [ ] 标题是否符合风格（简短+情绪化）
- [ ] 是否使用"老登/小登"等代称
- [ ] 句子是否够短、够口语
- [ ] 省略号是否自然出现
- [ ] 观点是否鲜明不模棱两可
- [ ] **新闻是否用过渡词串联，而非编号罗列**
- [ ] **是否融入了R9横纵分析法（纵向追溯+横向对比+横纵交汇）**
- [ ] **是否添加了深度历史复盘（如触发条件满足）**
- [ ] **所有数据是否经过校验**
- [ ] **是否通过内容安全审查（无高风险敏感词）**
- [ ] 结构是否完整（今日聚焦+重点新闻+全球市场）

## 输入示例

```
用户：今天A股三大指数都涨了，创业板涨最多2.7%，但是3900只个股下跌。
科技股内部软件跌硬件涨，半导体和CPO大涨。港股恒科涨4.7%，腾讯涨10%，
```

## 注意事项

1. **不编造事实**：所有行情数据、新闻事件必须基于用户提供的素材，不可 hallucinate
2. **代称适度**：网络化代称是风格核心，但首次出现某个代称时，建议读者能结合上下文理解
3. **观点有据**：调侃归调侃，分析逻辑必须自洽，不能为了风格牺牲专业性
4. **灵活调整**：如当日无重大事件，重点新闻可精简；如地缘/政策为主，可适当扩写
5. **默认不输出营销与合规内容**：文章末尾不添加任何投顾产品推广、"点赞转发"引导、以及基金合规声明。如用户明确要求添加此类内容，再按需附加
6. **历史对比不等于预测**：提供历史案例是为了帮助读者理解当前情境，不能简单线性外推"上次涨了这次也会涨"
